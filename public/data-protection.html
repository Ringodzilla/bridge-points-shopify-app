BridgePoint Data Protection Summary

Last updated: 2026-05-16

Purpose limitation

BridgePoint processes the minimum merchant and customer-linked data required to display store credit balances, create store credit transactions, generate operational logs, and process order-paid automatic grants.

Storage and retention

  • Store Credit balances remain in Shopify as the source of truth
  • BridgePoint stores only settings, operational logs, and duplicate-execution keys
  • BridgePoint requests protected customer fields only for Name and Email in v1
  • Data retention should be limited to operational, support, and compliance needs

Security controls

  • HTTPS/TLS is required in production
  • Production and test environments should remain separated
  • Access to personal data should be restricted to authorized operators only
  • Protected customer data requests should match the minimum fields actually used

Operational readiness

Merchants and reviewers can use this summary together with the privacy policy and release checklist to understand the app’s handling of protected customer data and operational risk.

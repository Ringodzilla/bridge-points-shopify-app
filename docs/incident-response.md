# BridgePoint Incident Response Policy

最終更新: 2026-04-29

## 1. 対象

以下の事象を対象とする。

- 不正アクセスの疑い
- 顧客データの漏えい、改ざん、消失の疑い
- billing や store credit の誤処理
- production 環境の停止

## 2. 初動

1. 影響範囲を確認する
2. 影響のある処理を停止または限定する
3. ログと監査証跡を保全する
4. 必要に応じてシークレットやアクセス権をローテーションする

## 3. 切り分け

- Shopify 側かアプリ側かを切り分ける
- customer data / billing / store credit のどれに影響するかを分類する
- 再発防止のため、根本原因と再現条件を記録する

## 4. 連絡

- マーチャント影響がある場合は、影響内容と暫定対応を速やかに共有する
- Shopify review や法令上必要な報告がある場合は、その要件に従う

## 5. 復旧

- 問題の修正後に再発確認を行う
- 必要なら billing や store credit への影響を監査する
- 復旧後に再発防止策を記録し、公開準備文書へ反映する
